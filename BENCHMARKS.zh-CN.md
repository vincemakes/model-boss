# Model Boss benchmarks（实测数据）

[English](BENCHMARKS.md) | **简体中文**

> 历史参考栈说明:这些运行测量的是 2026 年 Claude/Fable/Opus 组合,不能预测 Sol、Kimi 或未来模型 Profile 的节省幅度。`-42%` 与 `-89%` 是记录到的最强模型输出 token 变化;`-34%` 与 `-88%` 是按价格加权的最强模型额度变化,只作为额度代理。盲测 bug-hunt 只是一次单次观察的能力探针,不是普遍性证明。

> 发布来源说明:前身身份已归一化为中性标签,采集到的绝对工作区前缀也已归一化为 `<historical-workspace>`。记录的任务、闸门输出、结果以及 token/费用/时间测量值均未更改;该占位符不表示这些运行发生在 Model Boss 目录下。

这些是 Model Boss 继承的前身测量;Model Boss 本身并未运行这套语料。所有数字来自前身真实的 headless `claude -p` 运行,通过 CLI 的分模型用量 JSON 直接测得,零估算。每个运行都通过了全部闸门(`tsc --noEmit` + `vitest`)和全部质量断言;**各配置的产出质量完全一致**,差别只在费用、额度和时间。

## 测试方法

- 在继承的四项 eval 语料中,每个任务/配置组合只运行一次,都是独立的 headless `claude -p` 会话,跑在同一 fixture 仓库的全新拷贝里(真实 `pnpm` 项目,已 git 初始化)。
- 汇总中的 ± 表示四项不同任务之间的离散程度,不是重复试验方差,也不是统计置信度。
- 前身 skill 的 prompt 调用被测前身;baseline 禁用编排 skill,任务文本完全相同。
- 费用/token 来自 `--output-format json` 的分模型用量明细。
- 所有条件用完全相同的客观断言评分(重跑闸门、grep 验证、diff 范围检查)。
- 模型价格:Fable 5($10/$50 每百万 token)、Opus 4.8($5/$25)、Sonnet 5($3/$15)、Haiku 4.5($1/$5)。

## 前身结果:小任务反而更亏

三个小任务(≤ ~150 行改动、≤ 5 个文件):

| 任务 | Fable output(用前身) | Fable output(baseline) | Δ Fable | 总费用 用/baseline | 耗时 用/baseline |
|---|---|---|---|---|---|
| 多文件功能(模块 + 测试) | 4,277 | 2,757 | **+55%** | $1.25 / $0.78 | 121s / 53s |
| 全仓库机械改名 | 1,865 | 1,124 | **+66%** | $0.82 / $0.63 | 83s / 36s |
| API 错误信封设计+全量应用 | 6,415 | 4,798 | **+34%** | $1.85 / $1.08 | 235s / 84s |

在盈亏平衡点以下,"写任务包 + 审 diff"花掉的最强档 token 比直接把改动敲出来还多。质量断言两组各 30/30 全过——没有质量增益来抵消开销。Model Boss 的委托门槛(≥ ~300 行 / ≥ 6 文件 / 大量重复调用点)正是由这组实测数据定下的。

## 大任务:四种跑法全对照

同一个大体量任务——从零构建 8 模块购物车子系统(约 1,100 行代码 + 测试)——跑四种配置,全部闸门通过、断言满分(8/8 模块、测试全绿):

| 配置 | Fable output | Fable 费用 | 总费用 | 耗时 | 测试数 |
|---|---|---|---|---|---|
| 基线——Fable 全干 | 30,993 | $3.05 | $3.05 | 335s | 81 |
| **lite**——Fable 统筹,Sonnet 实现 | 17,899(−42%) | $2.03 | **$2.91(最低)** | 465s | **91** |
| **max**——Opus 4.8 主循环 + Fable 顾问 | **3,278(−89%)** | **$0.38** | $5.66 | **116s** | 83 |
| max 变体——Sonnet 5 主循环 | 12,152(−61%) | $1.41 | $5.87(最高) | 162s | 66 |

## Fable 额度账单,逐项列清

订阅限额计量的是**全维度加权用量**——输入、输出、缓存写、缓存读都算。同一任务里每种配置记在 Fable 名下的全部账目:

| 配置 | Input | Output | 缓存写 | 缓存读 | Fable 名下费用 | 额度消耗 vs 基线 |
|---|---|---|---|---|---|---|
| 基线 | 11,861 | 30,993 | 49,273 | 398,157 | $3.05 | — |
| lite | 11,595 | 17,899 | 40,451 | 205,528 | $2.03 | **−34%** |
| max(Opus 主循环) | 11,046 | 3,278 | 8,282 | **0** | $0.38 | **−88%** |
| max(Sonnet 主循环) | 22,614 | 12,152 | 41,098 | 65,919 | $1.41 | −54% |

官方没有公开额度加权公式,但单价对各维度的加权与算力成本同构——**"Fable 名下费用"这一列是额度消耗的最佳可用代理指标**,本项目所有"节省百分比"都锚定它。

注意 max-opus 那行缓存读的 **0**:顾问完全不携带会话上下文——两次无状态的"简报进、裁决出",与协议设计分毫不差。9 倍削减主要来自这里:在基线和 lite 里,最强模型光是*坐在主循环上*,每轮都要重读几十万缓存上下文 token。

## 能力探针:埋雷调试赛

上面的表只证明了*构建型*工作的成本差异。这个探针回答更难的问题:**把 Fable 移出主循环,会不会丢失判断密集型工作的解题能力?**

设置:一个计费聚合模块,埋 **6 颗生产级 bug**(半开区间双计费、逐行舍入漂移、内部引用泄漏、异步竞态丢更新、缓存聚合不失效、数字键字典序排序)。可见测试套件在 6 颗雷全在的情况下照样全绿。每种配置只拿到 4 份症状级 bug report(其中两份各混了两个根因)+ 行为规格——然后用它们**从未见过的盲测试集**判分,一颗雷一个针对性测试。

| | 基线——Fable 单干 | max——Opus 主循环 + Fable 顾问 |
|---|---|---|
| **盲测得分** | **6/6** | **6/6** |
| Fable 名下费用 | $1.58(干了全部活) | **$1.76(只当顾问,反而更多)** |
| 总费用 | **$1.58** | $3.45(+118%) |
| 耗时 | **152s** | 586s(3.9×) |

两个方向各一条结论:

- **能力持平是真的。** max 配置找到并修对了每一颗雷,包括竞态和引用别名这两颗最难的——根因推理能力在顾问架构下完整保留。
- **判断密集型工作上经济学反转。** 调试任务里*推理本身就是工作量*——没有便宜的实现体量可下放。顾问做的最强档推理($1.76)比 Fable 单干整个任务($1.58)还多,外面还要再付 Opus 和 Sonnet 的钱。此时编排是纯官僚成本:质量相同、费用 2.2 倍、时间 3.9 倍。这为 Model Boss 内建的两条规则——"调试不触发"、"设计密集型工作 orchestrator 亲自做"——提供了实测依据:当初是设计直觉,现在是数据。

(注:n=1 任务、每配置单次运行——这是能力探针,不是统计学结论。)

## 三条诚实结论

1. **max 档兑现了承诺——但只对额度。** Opus 主循环下 Fable 只花 $0.38(两次干净的顾问检查点),额度消耗降 8 倍;但总费用**反升 86%**:主循环税没消失,只是搬给了 Opus 且变大了(同样的统筹角色,Opus 输出 60k,Fable 只要 17.9k)。
2. **主循环模型的纪律比单价更重要。** Sonnet 5 单价只有 Opus 六成,却抱着 Fable 顾问不撒手(3.7 倍消耗)、cacheRead 冲到 460 万——额度和美元两头输。max 档推荐主循环:**Opus 级**。
3. **这个任务规模下,没有任何编排配置在总费用上赢过基线。** lite 最接近(−5%)。如果美元是你唯一的指标,编排只在远大于 1,100 行的任务上才可能回本。

## 你省的到底是什么?

两个完全不同的钱包:

- **按 API token 付费的用户** —— 总费用是唯一指标,而节省最多只有 −5%(lite)。费用差距随任务规模拉大——任务包+审查开销基本固定,下放的实现量随任务增长。只在真正的大任务上使用 Model Boss。
- **订阅制用户(Claude Max 等)** —— 真实约束是最强模型的**限额**。便宜档烧额度慢得多,而额度窗口到点重置。**最强档额度 −88% 意味着同一窗口能干约 8 倍的编排工作。** 这是 skill 的主要受众:首先是额度套利工具,其次才是省钱工具。
